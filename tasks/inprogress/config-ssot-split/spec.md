# Spec тАФ Split `.config` (local) vs `.config-remote` (SSOT)

Status: **in progress** ┬╖ Owner: LO ┬╖ Created: 2026-06-20

## 1. Goal

Two dotfile repos currently overlap. Resolve ownership so each has a single
responsibility:

- **`.config-remote`** = single source of truth + provisioner. Owns every
  *portable* config and the scripts that install/wire it. This is what runs on a
  fresh machine (laptop or cloud/SSH box) and sets everything up.
- **`.config`** = pure machine-local. Only configs that are meaningless on
  another machine (display, window manager, secrets, per-host overrides). Tracks
  **nothing** that `.config-remote` generates.

A config belongs in `.config-remote` if copying it to a new machine is correct.
It belongs in `.config` only if it is host-specific (GPU/display, WM layout,
local secrets).

## 2. Established pattern (must follow тАФ do not reinvent)

The refactor extends the existing idiom. New work mirrors it exactly.

### 2.1 `setup-<tool>.sh` тАФ idempotent, self-contained, single concern
- One file per tool/concern. Safe to re-run. Detects "already done" and skips.
- Cross-platform via `source .../git/lib.sh` тЖТ `detect_os` (`macos`/`debian`),
  `detect_arch` (`x86_64`/`arm64`), `require_brew`. Fail-fast on unsupported OS.
- **Install half**: apt on Debian, brew on macOS тАФ *or* deferred to the Brewfile
  on macOS (see ┬з2.4). **Wire half**: symlink or marker-block (┬з2.3).
- `set -euo pipefail`. `here="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`.

### 2.2 `run-setups.sh` тАФ per-domain orchestrator
- One per domain dir (`git/`, `terminal/`, `tmux/`, `md-rendering/`).
- `run() { echo "=== $1 ==="; "${here}/$1.sh"; }` then loop over setup names.
- Order-free (every setup is idempotent). apt-guards the install-only scripts;
  cross-platform wiring scripts always run. Pattern reference:
  `terminal/run-setups.sh`.

### 2.3 Wiring patterns (two, both idempotent)
- **Symlink**: home/XDG path тЖТ `.config-remote/<domain>/<file>`.
  Live examples: `~/.config/nvim тЖТ ../.config-remote/nvim`,
  `~/.tmux.conf тЖТ .config-remote/tmux/tmux.conf`,
  `~/.vimrc тЖТ .config-remote/vim/vimrc`.
- **Marker block**: inject one fenced block into an rc file, guarded by a
  `grep -qF` on the begin marker. Live example: `terminal/setup-shell.sh`
  appends `# >>> config-remote terminal snippets >>>` to `.zshrc`.

### 2.4 Entrypoints + package manifest
- `install-macos.sh` / `install-debian.sh` call each domain's `run-setups.sh`.
- **macOS packages are declarative**: `install-macos.sh:7` defers to
  `~/.config/brew/Brewfile` (+ `Brewfile.macos` casks). On Debian there is no
  Brewfile тАФ the `setup-*.sh` scripts are the installers.
- `lib.sh` is **sourced, never executed**.

## 3. Overlap catalogue (the problem)

### Already-clean тАФ the template to generalize
| concern | mechanism | owner |
|--|--|--|
| nvim | `~/.config/nvim` тЖТ symlink тЖТ `.config-remote/nvim` | remote тЬУ |
| tmux | `~/.tmux.conf` тЖТ symlink тЖТ `.config-remote/tmux/tmux.conf` | remote тЬУ |
| vim | `~/.vimrc` тЖТ symlink тЖТ `.config-remote/vim/vimrc` | remote тЬУ |
| zsh wiring | `setup-shell.sh` marker block in `.zshrc` | remote тЬУ |

### A. File tracked in `.config` AND generated by a `.config-remote` script
| concern | `.config` tracked file | `.config-remote` generator | now |
|--|--|--|--|
| global gitignore | `git/ignore` | `git/setup-globalignores.sh` appends missing patterns | in sync |
| lazygit pager | `lazygit/config.yml` | `git/setup-delta.sh` writes only if missing, warns if present | identical now |
| git delta/pager | `git/config` `[delta]`+`pager=delta` | `setup-delta.sh` installs delta, wires lazygit (not git/config) | seam: install remote, git wiring hand-held in `.config` |

> **DECISION (2026-06-20): keep these as seed-then-local-drift тАФ do NOT symlink.**
> The setup scripts are idempotent **seeders**, not enforcers: `setup-globalignores.sh`
> is append-only and `setup-delta.sh` writes the lazygit config only if absent (warns,
> never clobbers). The `.config` copy is **per-box state** that may intentionally diverge;
> re-running the setups from `.config-remote` reseeds a baseline without stomping local
> changes. Symlinking would forbid that, so it is rejected for this row. (Contrast: nvim/
> tmux/vim ARE symlinked because they carry no per-box variation.)

### B. Shell config sourced from BOTH repos into the same shell
Ordering: snippets block is appended at END of `.zshrc`, so `profile.d/*.sh`
runs AFTER the `.zshrc` body тЖТ **remote overrides local on every collision**.

| concern | `.config` | `.config-remote` | conflict |
|--|--|--|--|
| fzf init | `.zshrc:21` `source <(fzf --zsh)` + `fzf/config.zsh` (`:58`) | `profile.d/fzf.sh` (`eval $(fzf --zsh)` + all `FZF_*`) | fzf inited 2тАУ3├Ч; `FZF_DEFAULT_COMMAND` only in remote |
| tmux auto-attach | `.zshrc:9-11` + `.zprofile` | `profile.d/tmux-session.sh` `tmux()` wrapper | two mechanisms for session `main` |
| `rm`тЖТtrash | `.zshrc:54` `alias rm=trash` (mac) | `profile.d/trash.sh` `alias rm=trash-put` (Linux) | same alias, platform-split |
| EDITOR | unset | `profile.d/editor.sh` `export EDITOR=nvim` | env split across repos |
| ssh UX | `.zshrc:65-71` `ssh()` green bg | `profile.d/ssh-hints.sh` banner + `cd ~/lumigaia` | two behaviors; remote bakes machine path `~/lumigaia` |
| PATH | `.zshrc:39` `~/.local/bin` | `profile.d/fzf.sh` `~/.fzf/bin` | PATH assembled in both |

### C. Package install тАФ declarative vs imperative double-SSOT
| platform | `.config` | `.config-remote` |
|--|--|--|
| macOS | `brew/Brewfile` (+ `Brewfile.macos`) тАФ `install-macos.sh` defers to it | `terminal/setup-{bat,fd,fzf,ripgrep,trash}.sh`, `git/setup-lazygit.sh`, `setup-delta.sh`, `nvim/install-osx.sh` also brew-install same tools |
| Debian | тАФ (no Brewfile) | `install-debian.sh` тЖТ `setup-*.sh` are the sole installer |

тЖТ macOS installs each tool **twice**. Setup scripts' real macOS value is the
**wire**, not the install. Brewfile is **stale** (`duckdb` still listed; removed
casks never tracked).

### D. Stale orphan
| concern | active | orphan |
|--|--|--|
| vimrc | `~/.vimrc` тЖТ `.config-remote/vim/vimrc` | `.config/shell/.vimrc` тАФ tracked, linked to nothing. Dead. |

### E. Minor
`README.md`, `.gitignore`, `tasks/` exist in both (fine). Todo-tracking split
across both `tasks/` dirs.

## 4. Target ownership

**`.config-remote` (SSOT):** nvimтЬУ tmuxтЬУ vimтЬУ, **git config, lazygit config, fzf
config, all shell logic (`profile.d`)**, installers, **package manifest**.

**`.config` (pure local):** `ghostty` (GPU/display), `aerospace` (WM),
`espanso`, `devices`, `ides/.editorconfig`, `.zshrc.local`, secrets. Nothing the
remote generates.

## 5. Migration plan (in the repo idiom)

Each step is a `setup-*.sh` (symlink or marker-block) added under the right
domain, plus deletion of the now-duplicate tracked file in `.config`. Sequenced
safe-first.

1. **Delete dead orphan** тАФ remove `.config/shell/.vimrc` (active vimrc is the
   remote symlink). Zero risk.
2. **Fix stale Brewfiles** тАФ `Brewfile` drops `duckdb`; `Brewfile.macos` drops
   the removed casks (`gcloud-cli`, `monarch`, `warp`, `thebrowsercompany-dia`,
   `orbstack`, `tailscale-app` тАФ all uninstalled 2026-06-20). Final home/role of
   each Brewfile per the provisioning model (┬з6).
3. **fzf single owner** тАФ fold `.config/fzf/config.zsh` + `.zshrc:21` into
   `terminal/profile.d/fzf.sh`. Delete `.config/fzf/config.zsh` and the
   `.zshrc` fzf lines. fzf inits once.
4. **git config + ignore** тАФ KEEP as seed-then-local-drift (┬з3A decision). No
   symlink. `setup-globalignores.sh` stays append-only; `.config/git/{config,
   ignore}` remain per-box state. Only tidy: ensure the generator's baseline
   patterns stay a superset of what every box needs.
5. **lazygit config** тАФ KEEP as seed-then-local-drift (┬з3A decision). No symlink.
   `setup-delta.sh` already writes-if-missing / warns-if-present; leave it.
6. **Thin `.zshrc`** тАФ move `ssh()`, trash alias, tmux auto-attach out of the
   `.zshrc` body into `profile.d/` (dedupe vs existing snippets). `.zshrc` keeps
   only local bootstrap (brew shellenv, instant-prompt, p10k, plugin sources,
   `.zshrc.local`). Resolve the machine-specific `cd ~/lumigaia` in
   `ssh-hints.sh` (gate on hostname or move to `.zshrc.local`).
7. **Package install dedup** тАФ adopt the provisioning model (┬з6): single
   cross-platform manifests (`mise.toml` + `uv/tools.txt`) own installs; strip
   the *install* half from `setup-{bat,fd,fzf,ripgrep,trash,lazygit,delta}.sh`,
   leaving only the *wire* half. Brew demoted to mac casks + system libs.
8. **Regenerate wiring** тАФ each domain `run-setups.sh` re-run is idempotent;
   confirm a fresh-machine dry run.

## 6. Package provisioning model (mise + uv, brew demoted)

Today "what to install" is declared **twice** (Catalogue ┬зC): `Brewfile` on
macOS vs per-tool `setup-*.sh` on Debian. Replace both with **one
cross-platform manifest pair** that each OS reads identically. Brew is not
removed тАФ it is **demoted** from primary dev-tool installer to "macOS app +
system-lib layer."

### 6.1 Target home per tool

| layer | tools | home | mechanism |
|--|--|--|--|
| **Python CLIs** | ruff, httpie, trash-cli (Linux), basedpyright, mdformat | **`uv tool`** | extend existing `uv/tools.txt` + `bootstrap-uv.sh` |
| **Single static binaries** | bat, eza, fd, fzf, ripgrep, tealdeer, lazygit, lazydocker, tree-sitter-cli, gh, delta, lua-language-server, node, neovim | **mise** | one cross-platform `mise.toml`, version-pinned |
| **System / heavy libs** | tmux, ffmpeg, pandoc, vim, netbird | **apt (Linux) / brew (mac)** | codec deps, vendor repos, build pain тАФ keep |
| **GUI casks** | ghostty, obsidian, claude, aerospace, bitwarden, fonts, тАж | **brew cask (mac only)** | no alternative; untouched |

### 6.2 Why this shape
- **Kills ┬зC**: `mise.toml` + `uv/tools.txt` are read the same on mac and Debian
  тЖТ single SSOT for installs, version-pinned + reproducible. `setup-*.sh` shrink
  to **wire-only** (their real value).
- **uv-native**: you already provision Python tools from `uv/tools.txt`; ruff and
  httpie just move into that world.
- **Faster cloud provisioning**: ephemeral Debian boxes skip linuxbrew (~GB,
  slow, build deps) тАФ apt for system libs, mise for binaries, uv for Python.

### 6.3 Why brew stays (scope guard тАФ do NOT over-remove)
- Casks have no substitute; ffmpeg/system libs need a real dep resolver; mise is
  itself a new dependency that only pays off because Debian cloud boxes exist.
  Brew remains correct on macOS for casks + a handful of system tools.

### 6.4 Manifest design (repo idiom)
- `mise.toml` at `.config-remote/` root (or a `tools/` domain) тАФ declarative,
  like `Brewfile` but cross-platform.
- New `tools/setup-mise.sh` (bootstrap mise) + `tools/setup-uv-tools.sh` (wrap
  existing `bootstrap-uv.sh`), orchestrated by a `tools/run-setups.sh`. Same
  idempotent / `lib.sh` / `run()` pattern as every other domain.
- `install-{macos,debian}.sh` call `tools/run-setups.sh` early (binaries before
  wiring). macOS `install-macos.sh:7` note updated: packages come from
  `mise.toml` + `uv/tools.txt`, **not** the Brewfile; Brewfile narrows to casks.
- Brewfiles: `Brewfile` (mac system libs only тАФ tmux, ffmpeg, pandoc, vim,
  netbird) + `Brewfile.macos` (casks). Both move to `.config-remote` so a single
  repo provisions a machine.

### 6.5 Honest scope note
Pure infra. Payoff = faster, reproducible, version-pinned provisioning of cloud
research boxes + one manifest. Does **not** improve signal quality or research
velocity directly. Justified only as part of this SSOT task (it dissolves ┬зC);
not worth doing standalone.

## 7. Open questions
- **`.zshrc` ownership**: stays in `.config/shell/.zshrc` (local, symlinked from
  `~/.zshrc`) with remote contributing only the marker block? Or does the body
  also become remote-owned + symlinked? Leaning: thin local `.zshrc`, all
  portable logic in remote `profile.d/`.
- **git/config secrets**: `[user] email`, credential helper тАФ host-portable or
  per-host? Likely portable (same identity everywhere); confirm.
- **mise vs lighter `ubi`/`eget`**: mise adds version management + a daemon-ish
  shim layer. If pinning isn't valued, a thin `eget`-from-`lib.sh` fetch (the
  existing `setup-lazygit.sh` GitHub-release pattern, generalized) is lower-dep.
  Leaning mise for the single declarative manifest.

## 8. Validation (per step)
- After each step: open a fresh login shell; no errors, no double-sourced fzf
  (`type -a fzf`/check init runs once), `rm`тЖТtrash works, EDITOR=nvim.
- `git config --get core.pager` = delta; `git check-ignore -v .DS_Store` resolves
  via the symlinked ignore.
- Dry-run `install-macos.sh` on a scratch `$HOME` (or container for
  `install-debian.sh`) тЖТ every config lands via symlink/marker, no duplicate
  installs, idempotent on second run.
- `.config` `git ls-files` contains no remote-generated file.
